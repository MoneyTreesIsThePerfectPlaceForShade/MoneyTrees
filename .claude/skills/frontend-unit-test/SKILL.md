# Frontend Unit Test
## Инструкции
### Технологии
Модульные (они же `unit`) тесты на `TypeScript` файлы пишутся, используя `Jest`.
Модульные тесты на `tsx` файлы пишутся, используя `React Testing Library`.
### Подход к написанию тестов
#### Общие принципы
При написании тестов используют принцип `AAA`:
- Arrange - готовим данные;
- Act - выполняем действия;
- Assert - проверяем результат.

В качестве примера возьмём функцию умножения чисел:
```typescript
describe('Функция multiply', () => {
	it('вернёт 36 при умножении 12 на 3', ()=> {
		// Arrange
		const expectedResult = 36;
		// Act
		const actualResult = multiply(12,3);
		// Assert
		expect(actualResult).toBe(expectedResult);
	})
})
```

В модульном тесте используются структура из `describe` и `it`.
При этом, `describe` на первом уровне представляет собой тип тестируемого модуля и его название, а вложенный в него `it` представляет собой описание конкретного случая (тест-кейса).
Могут быть вложенные `describe`, если это уместно, но лучше избегать такой вложенности. Вложенные `describe` описывают обобщенный тест-кейс, а в нём лежат более конкретные тест-кейсы в `it`.
На первом уровне всегда один `describe`.

#### Множественная проверка в одном `it`
Для некоторых тестов удобно использовать `each`, чтобы проверить сразу множество случаев.

#### `Jest Hooks`
Все `Jest Hooks` используй в начале `describe`.

#### Моки
##### Моковые данные
Моковые данные пиши в отдельном файле и экспортируй их оттуда. А в тесте импортируй эти моки.
##### Моковые функции и шпионы
Всегда чисти моки функций и шпионов в `afterEach`, чтобы не было конфликтов в разных тестах.
Мокируй функции и создавай шпионов так, чтобы не приходилось делать каст к типу `jest.Mock`, если это возможно.

## Пример использования навыка
Тебя, как агента, попросили написать модульный (unit/юнит) тест на функцию, компонент, класс или файл.
Действуй по порядку:
1) Изучи функцию, компонент, класс или файл, на который необходимо написать модульные тесты;
2) Определи тест-кейсы, покрывая общие и краевые случаи;
3) Построй план написания модульных тестов;
4) Напиши модульные тесты;
## Список задач
Перед тем, как закончить работу, пройдись по списку задач и проверь, что выполнил все задачи:
- [ ] написаны тесты на все функции, методы, классы, компоненты или на всё, что было в файле, смотря какой был запрос;
- [ ] в тестах нет ошибок `TypeScript` и/или `ESLint`;
- [ ] если ошибки есть, то поправь их в соответствии с конфигурацией `TypeScript` и `ESLint`;
- [ ] тесты успешно проходят, нет упавших тестов;
- [ ] проверь, что покрытие тестами больше установленной планки.
